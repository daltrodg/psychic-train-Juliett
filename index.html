Sometimes web devs create user interfaces(UI) that are poorly designed and can make it difficult navigating or understanding a webpage. A lot of poorly designed UI's come in the flavor of forms. These forms can be used to create an account or enter a payment method. Some UI designs are just inefficient in the space they use or the way they capture the information you have entered. In this case of UI's that could use some improvment I wanted to look at Amazons payment method form.

Pictured above is what a user sees when they choose to enter a payment method onto Amazon's website. For the most part the UI is not bad and is more on the simplistic approach to UI. The problem is the unnecessary dropdown menus. In the payment method form, the expiration date input field includes two dropdown menus for a month and a year. There is one more dropdown menu that is used to link your card through your bank, if your bank is supported by this system. As of making this the system only supports five different banks. Why would they use a dropdown here? This is unnecessary clicking.

Above is the bank selection for linking a card through a bank. As you can see, only five banks are supported. These could have easily been displayed with buttons or radio buttons. If the link card button is pressed and no bank is selected it should instruct the user to select a bank and try again. The banks can be displayed as radio buttons or the logos of the banks can be used as buttons, displaying the name when hovering on the button.

Learn More

Here the bank selection section has been simplified to exclude the use of a dropdown and make the individual banks their own button for linking a card, there are only five banks so why not use buttons for this? This UI design employs the use of radio buttons to display all of the available banks. Dropdowns are useful is some areas of web development but for the most part, they are overused and not having the banks hidden behind a dropdown should be less tedious to a user.

Here the expiration date has been updated to exclude the use of dropdowns. The user will now type what number month the card expires and type the last two digits of the year, how it appears on the card. Realistically the only two numbers that are possible in the first two digits of the year is "20". We will not need the first two digits "30" for another few decades. We can worry about updating it to include the use of "30" when we need to because we need to make the user expirence a joy right now instead of complicating it for the future's sake.
